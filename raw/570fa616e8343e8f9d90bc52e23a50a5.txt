					<h1 class="articletitle">The DSA Toolkit Shines Light Into Dark and Stormy Archives</h1>
					<div class="abstract">
						<p>Themed web archive collections exist to make sense of archived web pages (mementos). Some collections contain hundreds of thousands of mementos. There are many collections about the same topic. Few collections on platforms like Archive-It include standardized metadata. Reviewing the documents in a single collection thus becomes an expensive proposition. Search engines help find individual documents but do not provide an overall understanding of each collection as a whole. Visitors need to be able to understand what individual collections contain so they can make decisions about individual collections and compare them to each other. The Dark and Stormy Archives (DSA) Project applies social media storytelling to a subset of a collection to facilitate collection understanding at a glance. As part of this work, we developed the DSA Toolkit, which helps archivists and visitors leverage this capability. As part of our recent International Internet Preservation Consortium (IIPC) grant, Los Alamos National Laboratory (LANL) and Old Dominion University (ODU) piloted the DSA toolkit with the National Library of Australia (NLA). Collectively we have made numerous improvements, from better handling of NLA mementos to native Linux installers to more approachable Web User Interfaces. Our goal is to make the DSA approachable for everyone so that end-users and archivists alike can apply social media storytelling to web archives.</p>

<h2>Web Archive Collections Are Too Large To Understand At A Glance</h2>
<p>Web archives are invaluable for a variety of research studies. <a href="https://us.sagepub.com/en-us/nam/the-sage-handbook-of-web-history/book252251" data-versionurl="https://archive.ph/wip/raZeC" data-versiondate="2022-04-11">Historians</a> have analyzed how humans interacted on extinct websites, like Geocities. <a href="https://doi.org/10.1002/meet.2011.14504801096" data-versionurl="https://arquivo.pt/wayback/20210528101030/https://asistdl.onlinelibrary.wiley.com/doi/full/10.1002/meet.2011.14504801096" data-versiondate="2021-05-28">Social scientists</a> have used them to study the changes in social commerce over time. Journalists can use web archive evidence to <a href="https://www.nytimes.com/2017/11/20/health/dermatology-skin-cancer.html" data-versionurl="https://web.archive.org/web/20220411200739/https://www.nytimes.com/2017/11/20/health/dermatology-skin-cancer.html" data-versiondate="2022-04-11">bring attention to questionable medical practices</a> and <a href="https://www.washingtonpost.com/politics/2020/04/03/jared-kushner-stands-trump-proceeds-offer-very-trumpian-claim-about-stockpiles/" data-versionurl="https://archive.ph/wip/WsPzq" data-versiondate="2022-04-11">document changes in government policy</a>.</p>
<p>Some archivists create themed <b>web archive collections</b> by selecting web pages for preservation that support a topic. Each web page, or <b>original resource</b>, can change over time. Archivists capture these original resources at specific points in time, turning each observation into a <b>memento</b>. The date and time of capture is that memento’s <b>memento-datetime</b>. A <b>TimeMap</b> contains the set of mementos for an original resource. <a href="https://archive-it.org" data-versionurl="https://archive.ph/wip/yjm2K" data-versiondate="2022-04-11">Archive-It</a> is a popular platform for creating themed web archive collections. Themed collections also exist at the <a href="https://www.loc.gov/web-archives/" data-versionurl="https://web.archive.org/web/20220411201202/https://www.loc.gov/web-archives/" data-versiondate="2022-04-11">Library of Congress</a>, <a href="https://conifer.rhizome.org/shawnmjones" data-versionurl="https://archive.ph/wip/Udghj" data-versiondate="2022-04-11">Conifer</a>, the <a href="https://haw.nsk.hr/en/" data-versionurl="https://archive.ph/wip/kAwC0" data-versiondate="2022-04-11">Croatian Web Archive</a>, the <a href="https://www.webarchive.org.uk/en/ukwa/collection" data-versionurl="https://web.archive.org/web/20220306195646/https://www.webarchive.org.uk/en/ukwa/collection" data-versiondate="2022-03-06">UK Web Archive</a>, and the National Library of Australia’s (NLA) <a href="https://pandora.nla.gov.au" data-versionurl="https://archive.ph/wip/pu6B7" data-versiondate="2022-04-11">PANDORA</a> and <a href="https://trove.nla.gov.au" data-versionurl="https://archive.ph/wip/sC5Ec" data-versiondate="2022-04-11">Trove</a> collections.</p>
<p class="caption"><a href="/media/issue53/jones/image-01.png"><img decoding="async" src="/media/issue53/jones/image-01_624.png" /></a><br />
<strong>Figure 1.</strong> Archive-It has two collections about the South Louisiana Flood in 2016.<br />
East Baton Rouge Parish Library created the <a href="https://archive-it.org/collections/7755" data-versionurl="https://archive.ph/wip/gfDjv" data-versiondate="2022-04-11">one represented by the left screenshot</a>.<br />
Old Dominion University created the one on the <a href="https://archive-it.org/collections/7760" data-versionurl="https://archive.ph/wip/7BB9B" data-versiondate="2022-04-11">right</a>. (Screenshots taken in June 2021.)</p>
<p>Such topical web archive collections are invaluable to those studying a topic, but it can be difficult for visitors to understand which collection they should begin exploring. Figure 1 shows screenshots of two Archive-It collections containing mementos about the South Louisiana Flood of 2016. Each collection contains different metadata fields. Which one should a researcher use in their project? Alternatively, if a visitor uses Archive-It’s collection-level search engine to find collections about a topic like “human rights,” they have 48 collections to review as of January 2022. Like <a href="https://archive-it.org/collections/3572" data-versionurl="https://archive.ph/wip/CsfCq" data-versiondate="2022-04-11"><i>Government of Canada Publications</i></a>, some collections contain hundreds of thousands of documents but no metadata to help visitors understand the collection. In a <a href="https://doi.org/10.1145/3357384.3358039" data-versionurl="https://archive.ph/wip/WzxTX" data-versiondate="2022-04-11">2019 study</a> (<a href="https://arxiv.org/abs/1905.11342" data-versionurl="https://archive.ph/wip/DcbyL" data-versiondate="2022-04-11"><i>preprint version</i></a>), we evaluated all collections at Archive-It and determined that collections with more original resources have less metadata to help visitors understand them.</p>
<p>Metadata is more consistent in the NLA’s PANDORA collections than in Archive-It’s collections, but the sheer size of some collections makes it difficult to understand them at a glance. Figure 2 shows a screenshot of the page for the <a href="https://pandora.nla.gov.au/subject/21" data-versionurl="https://web.archive.org/web/20220411202053/https://pandora.nla.gov.au/subject/21" data-versiondate="2022-04-11">PANDORA subject Politics</a>. This subject is a collection in its own right and contains subcategories and collections so that visitors can view parts of the collection. It also includes a listing of 5,269 page titles. At a minimum, each PANDORA collection contains a collection title and page titles. Some are divided into sub-collections, but a human would still need to review many documents (or at least page titles) to understand the collection.</p>
<p class="caption"><a href="/media/issue53/jones/image-02.png"><img decoding="async" src="/media/issue53/jones/image-02_624.png" /></a><br />
<strong>Figure 2.</strong> The PANDORA Subject <i>Politics</i> contains subcategories and collections, as well as a list of page titles.</p>

<h2>The DSA Toolkit For Summarizing Corpora</h2>
<p>The <a href="https://oduwsdl.github.io/dsa/" data-versionurl="https://archive.ph/wip/zsKnY" data-versiondate="2022-04-11">Dark and Stormy Archives (DSA) Project</a> helps visitors achieve collection understanding by finding solutions that summarize web archive collections through visualizations that provide understanding at a glance. We apply social media storytelling as that visualization because most visitors are already familiar with the paradigm and thus require no additional training to understand how to consume these visualizations.</p>
<p>Social media storytelling consists of <b>surrogates</b> that summarize individual pages. Figure 3 shows the same page rendered as a surrogate in different web platforms. Most web users understand how to interpret the cards from search results. Many readers understand how to interpret social cards used by Facebook, Twitter, Tumblr, and other platforms. These surrogates take different shapes, but each attempts to summarize an individual document.</p>
<p class="caption"><a href="/media/issue53/jones/image-03.png"><img decoding="async" src="/media/issue53/jones/image-03_624.png" /></a><br />
<strong>Figure 3.</strong> Different surrogates for the same web page rendered by different platforms.</p>
<p class="caption"><a href="/media/issue53/jones/image-04.png"><img decoding="async" src="/media/issue53/jones/image-04_624.png" /></a><br />
<strong>Figure 4.</strong> A social media story on Wakelet about the 2021 US Capitol Breach.</p>
<p class="caption"><a href="/media/issue53/jones/image-05.png"><img decoding="async" src="/media/issue53/jones/image-05_624.png" /></a><br />
<strong>Figure 5.</strong> A screenshot of the IIPC’s Archive-It Collection <i>Novel Coronavirus (COVID-19)</i>.</p>
<p>When we combine a group of surrogates together as a unit, we create a <b>story</b> that summarizes a <b>topic</b>. Figure 4 shows a story that summarizes the 2021 US Capitol Breach. This story contains surrogates, in the form of social cards, each describing a different web resource discussing the attack. Archive-It and PANDORA have their own surrogates. In Archive-It’s case, the surrogate consists of the URL captured and any metadata added by the archivist. In PANDORA’s case, each surrogate is a page title. We want to extend this idea of the social media story further than is already accomplished by these web archiving platforms.</p>
<p>Figure 5 shows a 2019 screenshot of the International Internet Preservation Consortium’s (IIPC) Archive-It Collection <a href="https://archive-it.org/collections/13529" data-versionurl="https://web.archive.org/web/20220411202240/https://archive-it.org/collections/13529" data-versiondate="2022-04-11"><i>Novel Coronavirus (COVID-19)</i></a>. This collection contains more than 23,000 mementos, far more than a single human can review, let alone understand at a glance. We applied the <a href="https://oduwsdl.github.io/dsa/software.html" data-versionurl="https://archive.ph/wip/UjYBQ" data-versiondate="2022-04-11">DSA toolkit</a> to produce a story summarizing this collection (Figure 6). The Archive-it collection contains metadata painstakingly provided by the archivist. We see page titles, countries of publication, languages, and, in some cases, descriptions of individual resources. There are even facets to help users explore different aspects of the collection. The story generated by the DSA Toolkit has people in masks, pictures of the virus, headlines, dates, sources, names, places, page summaries, maps showing the virus spreading across the world, and links back to the collection so visitors can explore that collection further.</p>
<p class="caption"><a href="/media/issue53/jones/image-06.png"><img decoding="async" src="/media/issue53/jones/image-06_624.png" /></a><br />
<strong>Figure 6.</strong> A story by the DSA Toolkit summarizing IIPC’s Archive-It Collection <i>Novel Coronavirus (COVID-19)</i> using sampling algorithms and social media storytelling.</p>
<p>The DSA Toolkit is built on <a href="https://doi.org/10.25777/zts6-v512" data-versionurl="https://archive.ph/wip/ergTn" data-versiondate="2022-04-11">our model of five storytelling processes</a>, as shown in Figure 7. A user follows these processes to tell a story that summarizes a corpus.</p>
<p class="caption"><a href="/media/issue53/jones/image-07.png"><img decoding="async" src="/media/issue53/jones/image-07_624.png" /></a><br />
<strong>Figure 7.</strong> The five processes for telling a story that summarizes a corpus.</p>
<p>As a summarization, our story provides a subset of the collection to a user. This subset consists of <b>exemplars</b> – documents that represent the collection well. Thus, we need to <b>select exemplars</b> from the collection to tell our story. Selecting exemplars can be done by humans or by various sampling algorithms. We can choose exemplars that attempt to summarize the collection as a whole or select those that feature a particular aspect of the collection, such as a specific time period, person, or source.</p>
<p>We <b>generate story metadata</b> to enrich our story for the visitor. Story metadata can consist of the collection name, who created the collection, and other collection metadata. The story from Figure 6 shows that story metadata can also contain entities, terms, images, and other content extracted from the collection.</p>
<p>Once our exemplars are selected, we <b>generate document metadata</b> to summarize each exemplar. In Figure 6, each of the social cards shown in the story visualizes this document metadata. Document metadata can take many forms, as needed by the storyteller.</p>
<p>We can then <b>visualize the story</b> in the desired medium by applying our story and document metadata. Most users will want a story that visitors can view on the web, and hence we have focused on providing visualizations that use HTML. We can also visualize stories in other media, such as video.</p>
<p>Finally, we can <b>distribute the story</b> for others to consume our visualization. Distribution can be from an author’s website. It could also be via social media. Alternatively, a user could potentially print the story and manually hand it to someone. The point is that visitors cannot consume a story if they cannot access it.</p>
<p>As shown in Figure 8, the DSA Toolkit provides tools to help meet the goals of this model. <a href="https://oduwsdl.github.io/hypercane/" data-versionurl="https://archive.ph/wip/6PUKa" data-versiondate="2022-04-11"><b>Hypercane</b></a> helps users select exemplars and generate story metadata. <a href="https://mementoembed.readthedocs.io/en/latest/" data-versionurl="https://web.archive.org/web/20220411202508/https://mementoembed.readthedocs.io/en/latest/" data-versiondate="2022-04-11"><b>MementoEmbed</b></a> generates document metadata. <a href="https://oduwsdl.github.io/raintale/" data-versionurl="https://archive.ph/wip/7zQGv" data-versiondate="2022-04-11"><b>Raintale</b></a> helps users visualize the story by accepting input from Hypercane and MementoEmbed. Not shown in Figure 8 are two additional supporting packages. The library <a href="https://github.com/oduwsdl/aiu" data-versionurl="https://web.archive.org/web/20220411202631/https://github.com/oduwsdl/aiu" data-versiondate="2022-04-11"><b>AIU</b></a> helps programmers identify the mementos and collection metadata from a web archive collection. The <a href="https://github.com/oduwsdl/off-topic-memento-toolkit" data-versionurl="https://archive.ph/wip/z2q3A" data-versiondate="2022-04-11"><b>OTMT</b></a> (short for Off-Topic Memento Toolkit) identifies off-topic mementos in a collection. This paper highlights these tools and the results of their recent pilot with the NLA, thanks to a <a href="https://netpreserve.org/projects/dark-and-stormy-archives/" data-versionurl="https://archive.ph/wip/2NKu1" data-versiondate="2022-04-11">grant</a> from the IIPC.</p>
<p class="caption"><a href="/media/issue53/jones/image-08.png"><img decoding="async" src="/media/issue53/jones/image-08_624.png" /></a><br />
<strong>Figure 8.</strong> How the DSA Toolkit fits with the storytelling model shown in Figure 7.</p>
<h2>Other Efforts For Understanding Web Archive Collections</h2>
<p>The DSA Toolkit is one of many tools for helping users understand web archive collections. For users with access to the Web ARChive (WARC) files that make up a web archive collection, <a href="https://github.com/helgeho/ArchiveSpark" data-versionurl="https://archive.ph/wip/VcolO" data-versiondate="2022-04-11">ArchiveSpark</a> (by <a href="http://doi.org/10.1145/2910896.2910902" data-versionurl="https://web.archive.org/web/20220411202936/https://dl.acm.org/doi/10.1145/2910896.2910902" data-versiondate="2022-04-11">Holzmann et al.</a>) and the <a href="https://archivesunleashed.org/aut/" data-versionurl="https://web.archive.org/web/20220331020941/https://archivesunleashed.org/aut/" data-versiondate="2022-04-11">Archives Unleashed Toolkit</a> (by <a href="https://doi.org/10.1145/3383583.3398513" data-versionurl="https://archive.ph/wip/tjuEl" data-versiondate="2022-04-11">Ruest et al.</a>) offer an environment for exploring collections.</p>
<p>A user can load WARCs into ArchiveSpark and then perform a set of operations for generating data, such as extracting titles, calculating term distributions, discovering named entities, building a graph of outgoing hyperlinks, and extracting images. ArchiveSpark typically outputs a JSON file consisting of the desired data. The output of one operation in ArchiveSpark can feed into another, allowing users to chain them together and produce the desired result. It is up to the end-user to further process this data with a third-party tool.</p>
<p>Archives Unleashed Toolkit (AUT) is built on <a href="https://github.com/lintool/warcbase" data-versionurl="https://archive.ph/wip/zzsIv" data-versiondate="2022-04-11">Warcbase</a> (<a href="https://doi.org/10.1145/3097570" data-versionurl="https://web.archive.org/web/20220411203249/https://dl.acm.org/doi/10.1145/3097570" data-versiondate="2022-04-11">Lin et al.</a>). Based on work with web archive researchers, Lin et al. developed four steps for those trying to work with web archives: filter, analyze, aggregate, and visualize. AUT implements these four steps. With AUT, a user can filter based on various features, such as original resource URL, language, domain name, memento-datetime, or URL pattern. An AUT user can also generate reports on top-level domains, named entities, or link structure. Archives Unleashed Cloud (AUK) integrates a subset of this functionality with Archive-It, providing different reports for archivists to apply to their own collections. If an AUT user wishes to visualize the output, they must use a third-party tool.</p>
<p>These tools inspired the development of the DSA Toolkit. Hypercane allows users to chain together several web archive collection operations, just like ArchiveSpark and AUT. Raintale accepts the output of Hypercane’s operations and visualizes it.</p>
<p>AUT and ArchiveSpark can potentially help archivists select exemplars and generate story metadata. They can also extract or generate some document metadata. Their focus, however, is not storytelling, so they rely on third-party tools for visualization and distribution.</p>
<p>The input type is another difference between the DSA Toolkit and these tools. Where ArchiveSpark and AUT accept WARCs as input, the DSA Toolkit does not. Because its goal is storytelling through summarization and visualization, the DSA Toolkit expects that all resources to be shared are accessible to the visitor of the story. Also, we wanted the DSA Toolkit to be usable by those who did not have access to a collection’s WARCs. Thus, the DSA Toolkit uses memento URLs instead of WARC files as its input.</p>
<p><a href="https://doi.org/10.1145/1367497.1367736" data-versionurl="https://web.archive.org/web/20220411203433/https://dl.acm.org/doi/10.1145/1367497.1367736" data-versiondate="2022-04-11">Jatowt et al.</a> may have been the first to visualize web pages over time. Their Page History Explorer downloaded mementos from web archives and generated screenshots ordered by memento-datetime as part of a timeline. Similarly, the <a href="https://tmvis.cs.odu.edu" data-versionurl="https://archive.ph/wip/vxitm" data-versiondate="2022-04-11">TMVis project</a> (by <a href="https://arxiv.org/abs/2006.02487" data-versionurl="https://archive.ph/wip/zHWv8" data-versiondate="2022-04-11">Mabe et al.</a>) accepts a single original resource URL, applies <a href="https://doi.org/10.1007/978-3-319-06028-6_25" data-versionurl="https://archive.ph/wip/2tfXz" data-versiondate="2022-04-11">AlSum’s Algorithm</a> to find the most novel mementos of that page, allows the user to reduce that set further manually, and then renders those mementos as a set of screenshots. These efforts work with a single original resource URL, not a collection of many different ones. However, they satisfy parts of our storytelling model by selecting exemplars, generating story metadata, generating document metadata (in the form of a screenshot), and visualizing the story – with Jatowt using a timeline and TMVis providing several different visual arrangements. With TMVis, users can distribute parts of their stories in various forms such as embeds, animated GIFs, and image sliders.</p>
<p><a href="https://doi.org/10.1145/2232817.2232821" data-versionurl="https://archive.ph/wip/J8yC4" data-versiondate="2022-04-11">Padia et al.</a> generated a set of visualizations for web archive collections. Padia’s visualizations did not focus on a single original resource URL but instead tried to consider different features of the mementos in the collection. They produced visualizations consisting of heat maps, timelines, word clouds, bubble charts, and treemaps. When possible, they leveraged the metadata provided by the collection, allowing them to cluster individual mementos as part of these visualizations. While they successfully created a set of visualizations to convey meaning about aspects of the collection, these visualizations required training for the viewer to understand them. Thus, they did not satisfy our goal of conveying meaning at a glance.</p>
<p class="caption"><a href="/media/issue53/jones/image-09.png"><img decoding="async" src="/media/issue53/jones/image-09_624.png" /></a><br />
<strong>Figure 9.</strong> With TMVis, users can explore a single original resource over time and visualize thumbnails for that resource.</p>
<p><a href="https://doi.org/10.1145/3091478.3091508" data-versionurl="https://web.archive.org/web/20220411203751/https://dl.acm.org/doi/10.1145/3091478.3091508" data-versiondate="2022-04-11">AlNoamany et al.</a> pioneered combining storytelling with web archive collections. At the time of her work, the most popular social media storytelling service was the <a href="https://web.archive.org/web/20220304092350/https://storify.com/faq-eol" data-originalurl="https://storify.com/faq-eol" data-versiondate="2022-03-04">now-defunct</a> Storify. She <a href="https://doi.org/10.1007/s00799-016-0185-3" data-versionurl="https://archive.ph/wip/4YKBe" data-versiondate="2022-04-15">analyzed Storify stories and compared them with Archive-It collections</a> to understand their similarities and differences. She identified that popular stories contain a median of 28 links, giving automated storytelling platforms a target number of items to select from a collection.</p>
<p>She identified some of the operations necessary to select exemplars automatically. Her algorithm filtered <a href="https://doi.org/10.1007/s00799-016-0183-5" data-versionurl="https://archive.ph/wip/9BDLH" data-versiondate="2022-04-11">off-topic</a> mementos, then duplicate mementos, and then non-English content from the collection. She also applied clustering, both by memento-datetime and by content. Finally, she scored the resulting mementos by Padia’s web page category, <a href="https://arxiv.org/abs/cs/0511077" data-versionurl="https://archive.ph/wip/Wbwqj" data-versiondate="2022-04-11">McCown’s URL path depth</a>, and Brunelle’s <a href="https://doi.org/10.1007/s00799-015-0140-8" data-versionurl="https://archive.ph/wip/Zjvso" data-versiondate="2022-04-11">memento damage</a>. Her algorithm, which the DSA Toolkit implements as <i>DSA1</i>, is the first of many possible methods for selecting exemplars. She <a href="https://doi.org/10.1145/3091478.3091508" data-versionurl="https://web.archive.org/web/20220411203751/https://dl.acm.org/doi/10.1145/3091478.3091508" data-versiondate="2022-04-11">demonstrated</a> (<a href="https://arxiv.org/abs/1705.06218" data-versionurl="https://archive.ph/wip/ldoGX" data-versiondate="2022-04-11"><i>preprint version</i></a>) that participants <i>could</i> determine the difference between randomly generated stories and those created by human archivists through a user study. Equally important, she confirmed that participants <i>could not</i> distinguish between stories produced by her visualization algorithm and those created by human archivists.</p>
<p>Figure 10 shows a story generated by AlNoamany’s proof-of-concept with Storify. Her story metadata was the collection name. Her proof-of-concept generated document metadata to fit into this platform. Storify handled distributing the story to visitors. Her research project was named <i>Dark and Stormy Archives</i>, and the current DSA project is a continuation of her work.</p>
<p class="caption"><a href="/media/issue53/jones/image-10.png"><img decoding="async" src="/media/issue53/jones/image-10_624.png" /></a><br />
<strong>Figure 10.</strong> A story produced by AlNoamany’s Dark and Stormy Archives proof-of-concept from the Archive-It collection <a href="https://archive-it.org/collections/2823" data-versionurl="https://archive.ph/wip/P319M" data-versiondate="2022-04-11"><i>Russia Plane Crash</i></a>.</p>

<h2>The DSA Toolkit</h2>
<p class="caption"><a href="/media/issue53/jones/image-11.png"><img decoding="async" src="/media/issue53/jones/image-11_624.png" /></a><br />
<strong>Figure 11.</strong> The DSA Toolkit workflow for producing a story relies heavily on Hypercane and Raintale.</p>
<p>The DSA Toolkit consists of five different software projects that satisfy our storytelling processes. Figure 11 provides an overview of the workflow for these tools. A user executes Hypercane to select exemplars and generate story metadata. Then they feed that output into Raintale to visualize their story. Both Hypercane and Raintale consult the other components as needed. Here we introduce each tool and highlight our recent improvements thanks to the pilot with NLA.</p>
<h3>AIU</h3>
<p>AIU (formerly Archive-It Utilities) is a Python library containing several classes that provide information about web archive collections. A programmer can invoke one of these classes and supply a collection identifier. The class includes several methods providing information like collection title, the URLs of the original resources in the collection, and other metadata, if present. AIU applies APIs and scraping HTML to gather its information.</p>
<p>AIU initially provided this information only for Archive-It collections. Archive-It collections contain metadata and a list of preserved original resources. From these original resources, we find TimeMaps. From these TimeMaps, we discover mementos. AIU follows this path to find the mementos in an Archive-It collection.</p>
<p>Below is an example iPython session where a user can extract information about an Archive-It collection (we measure the length of the list returned by the <code>list_seed_uris()</code> method for brevity, but it does produce a list of original resource URLs for the collection):</p>
<pre class="brush: plain; title: ; notranslate" title="">
In &#x5B;1]: from aiu import ArchiveItCollection

In &#x5B;2]: aic = ArchiveItCollection(5728)

In &#x5B;3]: aic.get_collection_name()
Out&#x5B;3]: 'Social Media'

In &#x5B;4]: aic.get_collectedby()
Out&#x5B;4]: 'Willamette University'

In &#x5B;5]: aic.get_description()
Out&#x5B;5]: 'Social media content created by Willamette University.'

In &#x5B;6]: aic.get_collection_uri()
Out&#x5B;6]: 'https://archive-it.org/collections/5728'

In &#x5B;7]: len(aic.list_seed_uris())
Out&#x5B;7]: 113
</pre>
<p>As part of our IIPC grant, we extended AIU to service the three types of collections at the NLA. as presented through their PANDORA and Trove websites. The PANDORA Archive is the curated selective web archive component of the larger Australian Web Archive (AWA). The NLA&#8217;s entire web archive corpus (including PANDORA) is accessible through the Trove discovery service. While the AWA includes PANDORA&#8217;s thematic sub-collections, the more extensive PANDORA subject listings are only viewable through the stand-alone PANDORA Archive website.</p>
<p>PANDORA subjects contain other PANDORA subjects listed as “subcategories.” Each PANDORA subject includes a list of page titles, as seen in Figure 2. Each page title represents an original resource and links to a Title Entry Page (TEP). The TEP serves as a TimeMap that links to all mementos for that original resource. AIU will follow this chain to find similar metadata to Archive-It, including original resource and memento URLs.</p>
<p>PANDORA subjects also contain PANDORA collections. Like subjects, each PANDORA collection can include collections and page titles. Each page title links to a TEP which functions the same as with PANDORA subjects. AIU will follow these links to find metadata and URLs for PANDORA collections.</p>
<p>Trove collections are slightly different. Each Trove collection fans out into subcollections. Subcollections contain direct links to mementos, as shown in Figure 12. Now AIU includes a class that will follow these links, gather metadata, and capture URLs as needed.</p>
<p>Please view <a href="https://github.com/oduwsdl/aiu/blob/master/README.md" data-versionurl="https://archive.ph/wip/9zSiB" data-versiondate="2022-04-11">AIU’s documentation</a> for more information on applying it to gather collection information.</p>
<p class="caption"><a href="/media/issue53/jones/image-12.png"><img decoding="async" src="/media/issue53/jones/image-12_624.png" /></a><br />
<strong>Figure 12.</strong> A screenshot of a Trove collection.</p>
<h3>OTMT</h3>
<p>The <a href="https://doi.org/10.17605/OSF.IO/UBW87" data-versionurl="https://archive.ph/wip/dKwln" data-versiondate="2022-04-11">Off-Topic Memento Toolkit</a> (OTMT) is an application and library that supports different text similarity measures for identifying which mementos in a collection are on-topic or off-topic. It is used as a library by Hypercane to perform this same task. The OTMT received bug fixes as part of our IIPC grant work.</p>
<h3>Hypercane</h3>
<p><a href="https://oduwsdl.github.io/hypercane/" data-versionurl="https://archive.ph/wip/6PUKa" data-versiondate="2022-04-11">Hypercane</a> is a complex application that provides the user with many ways to select exemplars and generate story metadata automatically. Hypercane exists in the command line application <code>hc</code>. A user selects exemplars with the <code>sample</code> action of that application. For example, to select exemplars from Archive-It collection 694 using the DSA1 algorithm, a user types:</p>
<pre class="brush: plain; title: ; notranslate" title="">
# hc sample dsa1 -i archiveit -a 694 -o story-mementos.tsv
</pre>
<p>Generating story metadata is handled by Hypercane’s <code>report</code> action. For example, to generate a list of named entities from a file containing a list of mementos, a user types:</p>
<pre class="brush: plain; title: ; notranslate" title="">
# hc report entities -i mementos -a story-mementos.tsv \
   -o entity-report.tsv
</pre>
<p>Finally, to create a rich story, a user can <code>synthesize</code> the output of other Hypercane commands into other formats. To create a rich JSON story file for Raintale, a user types:</p>
<pre class="brush: plain; title: ; notranslate" title="">
# hc synthesize raintale-story -i mementos -a story-mementos.tsv \
    --imagedata ${image_report} \
    --title &quot;My Story for Collection X&quot; \
    --termdata terms-report.tsv \
    --entitydata entity-report.tsv \
    -o mystory.json
</pre>
<p>The <code>sample</code>, <code>report</code>, and <code>synthesize</code> actions provide Hypercane’s top-level functionality. Their capabilities are not limited to what we show in the examples above. The <code>sample</code> action currently supports fourteen different sampling algorithms, denoted by their name – e.g., a user can execute <a href="https://en.wikipedia.org/wiki/Stratified_sampling" data-versionurl="https://archive.ph/wip/vMuBw" data-versiondate="2022-04-11">stratified random sampling</a> with <code>hc sample stratified-random</code>, or they can run the DSA3 algorithm from <a href="https://doi.org/10.25777/zts6-v512" data-versionurl="https://archive.ph/wip/ergTn" data-versiondate="2022-04-11">our previous work</a> with <code>hc sample dsa3</code>. Likewise, the <code>report</code> action supports ten different types of reports that can produce helpful metadata for storytelling, like extracted collection metadata, image analysis, or the list of top phrases for the input. The <code>synthesize</code> action gives the user different output format options, like a Raintale story file or WARCs.</p>
<p>As seen above, each Hypercane command supports the arguments <code>-i</code> (for input type) and <code>-a</code> (for input argument). The input type instructs Hypercane on the nature of the input argument. As seen above, if the input type is <code>archiveit</code>, then the input argument must be an Archive-It collection identifier. If the input type is <code>mementos</code>, then the input argument must be a file containing a list of memento URLs.</p>
<p>Hypercane currently supports the following input types:</p>
<ul>
<li aria-level="1"><code>mementos</code> &#8211; for a file containing a list of memento URLs</li>
<li aria-level="1"><code>timemaps</code> &#8211; the input argument is a file containing a list of TimeMap URLs</li>
<li aria-level="1"><code>original-resources</code> &#8211; the input argument is a file containing a list of original resource URLs</li>
<li aria-level="1"><code>archiveit</code> &#8211; the input argument is an Archive-It collection identifier</li>
<li aria-level="1"><code>pandora-subject</code> &#8211; the input argument is a PANDORA subject identifier, added as part of the IIPC grant work</li>
<li aria-level="1"><code>pandora-collection</code> &#8211; the input argument is PANDORA collection identifier, added as part of the IIPC grant work</li>
<li aria-level="1"><code>trove</code> &#8211; the input argument is a Trove collection identifier, added as part of the IIPC grant work</li>
</ul>
<p>Hypercane supports all Memento-compliant archives. Thus, a user need not rely upon mementos from Archive-It or NLA. The <code>mementos</code> and <code>timemaps</code> input types allow users to submit URLs from other archives, such as the Internet Archive’s Wayback Machine, Archive.Today, or Arquivo.pt.</p>
<p>Every Hypercane command also supports the <code>-o</code> argument to specify an output file. In most cases, the output file is a list of memento URLs. This feature allows the user to chain Hypercane commands together so that the output of one command can feed into another.</p>
<p>Hypercane’s sample action helps users execute existing sampling algorithms, but users also have access to the following algorithmic primitives that allow them to build their samples:</p>
<ul>
<li aria-level="1"><code>identify</code> &#8211; identifies the memento, TimeMap, or original-resource URLs for a given input type</li>
<li aria-level="1"><code>filter</code> &#8211; produces a list of mementos that match the provided criteria (e.g., are on-topic, have unique content, are written in a specific language)</li>
<li aria-level="1"><code>cluster</code> &#8211; clusters the collection via a given algorithm, such as through LDA topic modeling or K-means clustering by memento-datetime</li>
<li aria-level="1"><code>score</code> &#8211; scores each memento in the input by a given function</li>
<li aria-level="1"><code>order</code> &#8211; sorts the input by a given feature, like memento-datetime</li>
</ul>
<p>By combining these primitives, users can create powerful scripts for sampling from collections, allowing them to tell many types of stories. We will not describe all of these primitives in detail for brevity but instead, direct interested readers to Hypercane’s <a href="https://hypercane.readthedocs.io/en/latest/" data-versionurl="https://web.archive.org/web/20220411205145/https://hypercane.readthedocs.io/en/latest/" data-versiondate="2022-04-11">official documentation</a> and <a href="https://doi.org/10.1109/JCDL52503.2021.00049" data-versionurl="https://archive.ph/wip/AG4AD" data-versiondate="2022-04-11">our recent</a> <a href="https://doi.org/10.1145/3473044.3473047" data-versionurl="https://web.archive.org/web/20220411205351/https://dl.acm.org/doi/10.1145/3473044.3473047" data-versiondate="2022-04-11">publications</a>.</p>
<p>As part of the IIPC grant, we also sought to make Hypercane more approachable to administrators and end-users. Hypercane is a Python application and also depends on MongoDB for caching. We are evaluating native Linux installers for CentOS 8 and Ubuntu 21.10 that handle installing dependencies and provide convenience scripts for administering the application.</p>
<p class="caption"><a href="/media/issue53/jones/image-13.png"><img decoding="async" src="/media/issue53/jones/image-13_624.png" /></a><br />
<strong>Figure 13.</strong> A screenshot of Hypercane’s Web User Interface landing page.</p>
<p>We developed a new Web User Interface (WUI) so that Hypercane was approachable for users not familiar with scripting or working with command-line interfaces. Figure 13 shows a screenshot of this WUI. We wanted to create a web user interface while still preserving the existence and capabilities of the command-line application for scripting. We built Hypercane’s WUI with the <a href="https://wooey.readthedocs.io/en/latest/" data-versionurl="https://web.archive.org/web/20220411205446/https://wooey.readthedocs.io/en/latest/" data-versiondate="2022-04-11">Wooey</a> library, which creates web user interfaces from command-line applications, satisfying our needs. With Wooey, a user can fill out a web form to submit a Hypercane command as a job and come back later to retrieve the results. Wooey also provides authentication and separation of jobs to protect users’ privacy as a Django application.</p>
<p>Hypercane’s <code>sample</code> action allows users to run pre-existing sampling algorithms to get a list of mementos for their story. As noted above, it supports ten sampling algorithms, including AlNoamany’s. But what if a user wants to run <code>sample</code>, execute all <code>report</code> actions necessary to generate different types of story metadata, and finally <code>synthesize</code> a JSON file for use in a rich Raintale story? Before the WUI, we suggested that they write a script that executes these commands in order. The WUI presents <b>convenience recipes</b> that help users run complex tasks with minimal input and no scripting. Hypercane’s WUI currently supports seven convenience recipes as web forms.</p>
<p class="caption"><a href="/media/issue53/jones/image-14.png"><img decoding="async" src="/media/issue53/jones/image-14_624.png" /></a><br />
<strong>Figure 14.</strong> A screenshot of the Hypercane web form for the recipe “Summarize a collection as a Raintale story file.”</p>
<p class="caption"><a href="/media/issue53/jones/image-15.png"><img decoding="async" src="/media/issue53/jones/image-15_624.png" /></a><br />
<strong>Figure 15.</strong> A screenshot of a completed Hypercane job. The user can click the Download button (top right) and save the Raintale story file. (The red text only indicates a warning.)</p>
<p>Most users just wish to leverage Hypercane’s storytelling capabilities. The recipe “Summarize a collection as a Raintale story file” accepts a collection ID for Archive-It, PANDORA Subject, PANDORA Collection, or Trove and executes the necessary <code>sample</code>, <code>report</code>, and <code>synthesize</code> commands to produce a Raintale story file. A user will need to supply the collection type, collection ID, sampling algorithm, and story title, as shown in Figure 14. Figure 15 shows the interface once the job is complete, allowing the user to download their Raintale story file.</code></code></p>
<p>Some users already have a list of memento URLs or wish to use mementos from a Memento-compliant archive not listed above. The recipe “Summarize archived pages as a Raintale story file” accepts an input file containing a newline-separated list of memento URLs and does the same as the recipe above.</p>
<p>Other users have already selected exemplars from web archives and do not need to execute the <code>sample</code> step. The recipe “Create a Raintale story file from archived page URLs” accepts an input file containing a newline-separated list of memento URLs and executes the necessary <code>report</code> and <code>synthesize</code> commands to produce a Raintale story file.</p>
<p>Sometimes archivists or researchers just need a list of the memento page URLs in a collection. The “Generate archived page URLs from a collection” recipe accepts a collection ID for Archive-It, PANDORA Subject, PANDORA Collection, or Trove and executes the necessary <code>identify</code> command to find all memento page URLs in the collection. Similarly, a user may want the list of original resource URLs in a collection. The “Generate original page URLs from a collection” recipe performs the same steps but outputs a list of original resource URLs. The user could also run the “identify by Collection ID” web form, but these recipes present simplified language and options.</p>
<p>Additionally, there are cases where a user will desire WARCs for a third-party application. The recipe “Synthesize WARCs from a collection” accepts a collection ID and will execute the necessary <code>identify</code> and <code>synthesize</code> steps to convert the mementos in the collection into WARCs that closely resemble what the archive initially crawled. Hypercane leverages the Memento protocol and its knowledge of accessing <a href="https://ws-dl.blogspot.com/2016/08/2016-08-15-mementos-in-raw-take-two.html" data-versionurl="https://archive.ph/wip/RORDI" data-versiondate="2022-04-11">raw memento</a> content from different web archives to make this happen. These WARCs are an approximation built from what Hypercane can access via the web and are not identical to the web archive’s holdings. Still, they are sufficient for experimentation and analysis with tools like AUT. Likewise, a user can produce WARCs from a list of memento URLs with the recipe “Synthesize WARCs from archive page URLs.”</p>
<p>These recipes continue to make Hypercane more approachable for end-users. Though Hypercane primarily exists to feed content to Raintale, it has evolved into a much more promising toolkit of its own.</p>
<h3>MementoEmbed</h3>
<p>MementoEmbed is a surrogate generation tool that can create four different types of surrogates. It produces social cards like those seen on Facebook and Twitter, page screenshots (also called browser thumbnails), word clouds, and <b>imagereels</b>. Imagereels are animated GIFs (GIF version 89a) of the top images found in a memento. Figure 16 displays one of MementoEmbed’s social cards.</p>
<p>MementoEmbed also supports a web API for generating document metadata. Through this web API, a client can create surrogates. Clients can also request specific document metadata, such as page titles, memento-datetimes, image ranking, and automatic text summaries.</p>
<p>We developed MementoEmbed after <a href="https://ws-dl.blogspot.com/2017/08/2017-08-11-where-can-we-post-stories.html" data-versionurl="https://archive.ph/wip/YOLEd" data-versiondate="2022-04-11">analyzing more than 50 platforms</a> and finding that none support mementos correctly. Some platforms do not understand how to extract the original resource URL from the memento. Others could not differentiate between a web archive’s content and the memento content. Still, others refused to generate surrogates for mementos.</p>
<p>MementoEmbed is critical to the DSA Toolkit. Hypercane applies MementoEmbed’s functions that <a href="https://doi.org/10.1145/3447535.3462505" data-versionurl="https://archive.ph/wip/2nXyh" data-versiondate="2022-04-11">discover images</a> (<a href="https://arxiv.org/abs/2103.04899" data-versionurl="https://archive.ph/wip/wbV0p" data-versiondate="2022-04-11"><i>preprint version</i></a>) and raw memento content. Raintale is a <a href="https://arxiv.org/abs/2008.00137" data-versionurl="https://archive.ph/wip/tcqLX" data-versiondate="2022-04-11">client</a> of MementoEmbed’s web API.</p>
<p>Most web archives augment their mementos with navigation elements and branding to aid visitors. Unfortunately, these augmentations confuse natural language processing technology, like that implemented by Hypercane and MementoEmbed. For example, when comparing frequent terms among a set of mementos, the word “Trove” appears to be the most frequent word, even though it does not represent the actual memento content. For Wayback and OpenWayback web archives, like Archive-It, MementoEmbed simply inserts an <code>id_</code> flag into their memento URL to visit a page lacking these augmentations. Trove mementos required different handling.</p>
<p>Figure 17a displays a screenshot of a Trove memento. Figure 17b blurs its augmentations to emphasize the actual memento’s content. As part of our pilot with the NLA, we had to update MementoEmbed to discard these augmentations when processing Trove’s mementos. Finding the actual memento content required the following steps:</p>
<ol>
<li aria-level="1">change the domain name in the memento URL from webarchive.nla.gov.au to web.archive.org.au</li>
<li aria-level="1">insert the <code>id_</code> flag into the memento URL</li>
</ol>
<p>From here, MementoEmbed can now process the raw memento content.</p>
<p class="caption"><a href="/media/issue53/jones/image-16.png"><img decoding="async" src="/media/issue53/jones/image-16_624.png" /></a><br />
<strong>Figure 16.</strong> A screenshot of a MementoEmbed social card containing a striking image, title, description, original resource domain, and favicons, as well as information about the archive, when the archive preserved the page, and links to other mementos.</p>

<p>We also updated MementoEmbed to better handle creating screenshot surrogates of Trove’s mementos. The additional augmentations required changing the timeouts on MementoEmbed’s screenshot capabilities.</p>
<p>As with Hypercane, we created native Linux installers for MementoEmbed for CentOS 8 and Ubuntu 21.10 to help system administrators quickly stand up MementoEmbed. For more information on MementoEmbed, please <a href="https://mementoembed.readthedocs.io/en/latest/" data-versionurl="https://web.archive.org/web/20220411202508/https://mementoembed.readthedocs.io/en/latest/" data-versiondate="2022-04-11">see its documentation</a>.</p>
<h3>Raintale</h3>
<p>Hypercane focuses on selecting exemplars and generating story metadata. MementoEmbed focuses on generating document metadata for a single memento. Raintale takes in all of this information and produces a story in a format that the storyteller desires. Raintale is a command-line application executed by the <code>tellstory</code> command. To apply the rich story file generated in the Hypercane section above, a user types:</p>
<pre class="brush: plain; title: ; notranslate" title="">
# tellstory -i mystory.json --storyteller template \
    --story-template mytemplate.html -o mystory.html
</pre>
<p>The <code>-i</code> argument specifies the input file containing the information for the story. The <code>--storyteller</code> argument indicates the type of story being told. Because the above example suggests that we wish to use a template, the <code>--story-template</code> argument is needed to specify a file containing a template that Raintale will use to format the story. Finally, the <code>-o</code> argument indicates where to store the resulting story.</p>
<p>The user heavily controls the format of Raintale stories through templates. Figure 18 shows a story that applies a Bootstrap carousel template to a set of mementos to summarize the Trove collection <a href="https://webarchive.nla.gov.au/collection/13524" data-versionurl="https://archive.ph/wip/xcPx0" data-versiondate="2022-04-11"><i>Tourism</i></a>. Figure 18 shows a story that applies an NLA-authored template to a group of hand-selected mementos about Australian Zoos, Wildlife Sanctuaries &amp; Aquariums.</p>
<p class="caption"><img decoding="async" src="/media/issue53/jones/image-18.gif" /><br />
<strong>Figure 18.</strong> A Raintale story generated using a Bootstrap carousel.</p>
<p class="caption"><a href="/media/issue53/jones/image-19.png"><img decoding="async" src="/media/issue53/jones/image-19_624.png" /></a><br />
<strong>Figure 19.</strong> A Raintale story generated using a template developed by the National Library of Australia.</p>
<p>Because Raintale supports templates, users can construct stories that take many forms, from the COVID-19 story in Figure 6 to the stories shown in Figures 17 and 18. Raintale applies a <a href="https://raintale.readthedocs.io/en/latest/templates.html" data-versionurl="https://archive.ph/wip/AkOz5" data-versiondate="2022-04-11">modified version of the Jinja2 template language</a> that supports additional options for extracting specific document metadata for the output. This template language does not limit Raintale to HTML. It permits many textual formats, like Markdown, XML, JSON, and Jekyll.</p>
<p>As with Hypercane, we recognized that a command-line application is not always approachable for all software users. As part of our pilot, we leveraged Wooey to create a WUI for Raintale as well. Figures 20–22 show screenshots of a user executing “Create Story From Template,” which produces a templated story just like we demonstrated with the <code>tellstory</code> command above. The “Create Story From Preset” option lets users forgo a template in favor of built-in story templates. With “Tell Story With Twitter,” Raintale will create a Twitter thread from the mementos in the input. Finally, “Create Video Story” generates an MP4 file consisting of the top-ranked images and text from each memento in the story.</p>
<p class="caption"><a href="/media/issue53/jones/image-20.png"><img decoding="async" src="/media/issue53/jones/image-20_624.png" /></a><br />
<strong>Figure 20.</strong> A screenshot of the landing page of the Raintale WUI.</p>
<p class="caption"><a href="/media/issue53/jones/image-21.png"><img decoding="async" src="/media/issue53/jones/image-21_624.png" /></a><br />
<strong>Figure 21.</strong> A screenshot of the web form allowing a user to “Create Story From Template.”</p>
<p class="caption"><a href="/media/issue53/jones/image-22.png"><img decoding="async" src="/media/issue53/jones/image-22_624.png" /></a><br />
<strong>Figure 22.</strong> A screenshot showing a completed Raintale job where the user can download the story.</p>
<p>As with Hypercane and MementoEmbed, we developed native CentOS 8 and Ubuntu 21.10 installers for Raintale. We presented Raintale at the <a href="https://fox.cs.vt.edu/wadl2020.html" data-versionurl="https://archive.ph/wip/TprQV" data-versiondate="2022-04-11">WADL 2020</a> workshop and <a href="https://netpreserve.org/ga2021/" data-versionurl="https://archive.ph/wip/xWLul" data-versiondate="2022-04-11">IIPC WAC 2021</a>. An early version of Raintale helped us conduct user testing that <a href="https://doi.org/10.1145/3357384.3358039" data-versionurl="https://archive.ph/wip/WzxTX" data-versiondate="2022-04-11">showed that social cards probably provide a better understanding of web archive collections</a> when compared to alternatives like page screenshots. For more information on installing and leveraging Raintale, please consult <a href="https://raintale.readthedocs.io/en/latest/index.html" data-versionurl="https://web.archive.org/web/20220411211016/https://raintale.readthedocs.io/en/latest/index.html" data-versiondate="2022-04-11">its documentation</a>.</p>
<h2>The Future of Storytelling With Web Archives</h2>
<p>We see a lot of potential for further development of these tools. This pilot project has helped us better understand the needs of our users. We continue to improve the error handling of these tools as we learn more from the community. Overall, we want to improve the approachability of these tools.</p>
<p>Our native installers and Web User Interfaces are a start but only apply to Linux. Thanks to the experience of creating an installer for Linux, we have some ideas for a macOS DMG file that supports the applications of Hypercane, MementoEmbed, and Raintale.</p>
<p>With Wooey, we could preserve our existing command-line interfaces for scripting while also providing new web user interfaces. The authors of <a href="https://pypi.org/project/Gooey/" data-versionurl="https://web.archive.org/web/20220411211150/https://pypi.org/project/Gooey/" data-versiondate="2022-04-11">Gooey</a> promise this same functionality using the same methods but with a native non-browser graphic user interface. We are hopeful that our efforts with Wooey might make it easier to apply Gooey and provide a proper native GUI interface.</p>
<p>As we build upon these technologies, we are charting a path toward making the DSA Toolkit a set of Microsoft Windows desktop applications. As a set of Python applications, the DSA Toolkit will have to contend with library installation issues on Windows. Once we resolve those issues, we will have reached the maximum number of users with the current code.</p>
<p>We are also exploring additional functionality. AIU, and, by proxy, Hypercane now support Archive-It, Trove, and PANDORA. We can expand it to support web archive collections from the Croatian Web Archive, the UK Web Archive, the Library of Congress, and Conifer. This way, users can apply our tools to summarize more collection types. Of course, we would continue to support sets of memento URLs as input because that would extend the reach to web archives that do not currently support collections.</p>
<p>The Hypercane recipes inspired by our collaboration are just the beginning. We are exploring the idea of a “Recipe Builder” where users can create their own Hypercane scripts but in a much more user-friendly fashion than shell scripting. Once this is in place, we hope that users will share their recipes. We could even facilitate a recipe exchange for interested archives that want to share Hypercane solutions.</p>
<p>Raintale supports templates so that archivists can customize their stories. We see a future where archivists can share the look and feel of their stories with each other. In addition to templates, we are considering the idea of a “Story Builder” allowing users to create stories graphically in real-time, much like bloggers do with WordPress blocks.</p>
<p>Our collaboration between LANL, ODU, and the NLA has been fruitful but was only the beginning. Either by executing these tools on your own or visiting someone else’s stories, what gems will you discover in a web archive collection?</p>
